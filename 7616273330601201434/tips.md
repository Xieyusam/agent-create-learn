### 8. 安全验证与代码质量

#### 8.1 哈希函数使用规范

- **禁止使用 MD5 和 SHA1**：这些算法已被认为不安全，容易遭受碰撞攻击。
- **推荐使用的哈希算法**：
  - `blake2b` / `blake2s`：现代化、高性能、安全的哈希函数
  - `sha256` / `sha512`：如需兼容性可使用 SHA-2 系列
- **非安全用途的哈希**：如果哈希仅用于生成 ID、分桶、缓存键等非安全目的：
  - 优先使用 `blake2b`（速度快且安全）
  - 在代码中添加明确注释说明用途
  - 示例：
    ```python
    # 使用 blake2b 用于非安全目的（ID 生成和分桶）
    hashlib.blake2b(value.encode("utf-8"), digest_size=16).hexdigest()
    ```

#### 8.2 Bandit 安全扫描配置

- 项目应包含 `.bandit` 配置文件，明确排除误报和合理的安全例外。
- 常见需要配置的检查项：
  - `B303`：MD5 使用检查（如已替换为 blake2b 可跳过）
  - `B324`：hashlib 非安全用途检查
- 配置示例：
  ```ini
  [bandit]
  exclude_dirs = /tests/,/venv/,/.venv/,/env/
  skips = B303,B324
  ```

在提交代码前，应确保通过以下检查：

- **测试覆盖率**：确保核心功能有充分的单元测试覆盖

#### 8.4 常见安全问题规避

- **SQL 注入**：使用参数化查询，不要拼接 SQL 字符串
- **命令注入**：避免使用 `os.system()`、`subprocess.shell=True`
- **路径遍历**：验证用户输入的文件路径，使用 `Path.resolve()` 规范化
- **敏感信息泄露**：
  - 不在日志中输出完整的 API Key
  - 异常信息中不包含敏感配置
  - 使用 `***` 或截断方式显示密钥（如 `sk-***abc`）
